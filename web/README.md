# Web Programming Labs 

В этой папке собраны лабораторные работы по курсу веб-программирования в Университете ИТМО.

## Описание

Лабораторные работы в этой директории отражают последовательное развитие одного и того же класса задач: проверка попадания точки в область на координатной плоскости. От лабораторной к лабораторной усложняются архитектура приложения, стек технологий, способ организации клиентской и серверной логики, хранение данных и требования к пользовательскому интерфейсу.

Если в начале курс концентрируется на базовых веб-технологиях и ручной организации HTTP-взаимодействия, то далее происходит переход к серверным Java-технологиям, шаблону MVC, компонентным фреймворкам, ORM, REST API и полноценному SPA-интерфейсу.

## Структура

	•	lab1/ — FastCGI-сервер на Java, Apache httpd, HTML/CSS/JavaScript, AJAX
	•	lab2/ — Java Servlets, JSP, MVC, WildFly
	•	lab3/ — JSF, managed beans, Facelets, JPA, PostgreSQL
	•	lab4/ — Java EE / EJB backend, REST API, React + Redux frontend, JPA

## Эволюция технологий по лабораторным работам

### Lab 1

Первая лабораторная работа посвящена базовым принципам веб-разработки и взаимодействию клиента с сервером.

Используемые технологии:

	•	HTML
	•	CSS
	•	JavaScript
	•	AJAX
	•	Java FastCGI
	•	Apache httpd

Основная задача:

	•	разработка HTML-страницы, которая отправляет координаты точки и параметр радиуса на сервер;
	•	реализация FastCGI-сервера на Java, который выполняет проверку попадания точки в область;
	•	возврат результата проверки, времени обработки и текущего времени;
	•	сохранение и отображение истории предыдущих запросов.

Что добавляется и отрабатывается:

	•	ручная работа с HTTP-запросами;
	•	разделение статического и динамического контента;
	•	настройка Apache httpd;
	•	серверная обработка параметров;
	•	клиентская валидация формы на JavaScript;
	•	использование AJAX для асинхронного обращения к серверу;
	•	базовая работа с DOM и таблицей результатов;
	•	понимание роли веб-сервера и отдельного backend-процесса.

### Lab 2

Во второй лабораторной работе приложение переносится на платформу Java EE c использованием сервлетов и JSP. Появляется более явная серверная архитектура и шаблон MVC.

Используемые технологии:

	•	Java Servlets
	•	JSP
	•	JavaScript
	•	WildFly
	•	MVC

Что появляется по сравнению с Lab 1:

	•	вместо FastCGI используется servlet-based приложение;
	•	запросы централизуются через ControllerServlet;
	•	логика проверки выносится в AreaCheckServlet;
	•	пользовательский интерфейс формируется через JSP;
	•	приложение разворачивается в контейнере сервлетов WildFly;
	•	результаты предыдущих проверок сохраняются в контексте приложения, сессии или bean-компоненте.

Новые фичи и архитектурные изменения:

	•	централизованная маршрутизация запросов;
	•	переход от “страница + внешний сервер” к полноценному Java web application;
	•	применение шаблона MVC;
	•	разделение обязанностей между контроллером, обработчиком логики и представлением;
	•	использование JSP для генерации HTML;
	•	поддержка интерактивной картинки, по клику на которую вычисляются координаты точки;
	•	динамическое обновление изображения с отмеченными точками;
	•	более формальная работа с состоянием пользователя через HTTP-сессию и server-side storage.

### Lab 3

Третья лабораторная работа переводит приложение на компонентный серверный фреймворк JavaServer Faces. На этом этапе усложняется модель UI, появляется ORM и база данных PostgreSQL.

Используемые технологии:

	•	JavaServer Faces (JSF)
	•	Facelets
	•	Managed Beans
	•	JPA
	•	PostgreSQL
	•	PrimeFaces / ICEfaces
	•	WildFly или другой Java EE совместимый сервер

Что добавляется по сравнению с Lab 2:

	•	переход от сервлетов и JSP к компонентной модели JSF;
	•	использование facelets-шаблонов вместо JSP-представлений;
	•	серверная логика переносится в managed beans;
	•	конфигурация бинов задаётся аннотациями;
	•	результаты проверки теперь сохраняются в PostgreSQL;
	•	доступ к данным организуется через JPA и ORM-провайдер;
	•	навигация между страницами задаётся отдельным конфигурационным файлом.

Новые фичи:

	•	стартовая и основная страницы как отдельные шаблоны;
	•	интерактивные часы на стартовой странице;
	•	динамически обновляемая картинка области;
	•	автоматическая перерисовка области при изменении радиуса;
	•	таблица результатов, связанная с managed bean;
	•	цветовая индикация попадания / непопадания;
	•	server-side валидация и конвертация данных на уровне JSF-компонентов;
	•	переход от временного хранения данных к постоянному хранению в БД.

По сути, Lab 3 — это переход от классического server-side MVC к декларативному компонентному UI и ORM-модели доступа к данным.

### Lab 4

Четвёртая лабораторная работа — это уже полноценное разделение frontend и backend. Приложение из предыдущей лабораторной переписывается в виде SPA-клиента на React и серверной части на Java EE с использованием EJB и REST API.

Используемые технологии:

	•	Java EE
	•	EJB
	•	JPA
	•	REST API
	•	React
	•	Redux
	•	JSX / ES6
	•	PrimeReact
	•	HSQLDB

Что добавляется по сравнению с Lab 3:

	•	frontend и backend разделяются на независимые уровни;
	•	сервер больше не рендерит HTML-страницы приложения;
	•	клиентский интерфейс реализуется как React-приложение;
	•	обмен данными выполняется через REST API;
	•	состояние интерфейса управляется через Redux;
	•	добавляется аутентификация пользователя;
	•	появляется отдельная таблица пользователей;
	•	пароли хранятся в виде хэш-сумм;
	•	доступ к основной странице ограничивается авторизацией;
	•	интерфейс адаптируется под desktop, tablet и mobile;
	•	backend строится на EJB-компонентах;
	•	данные продолжают храниться через JPA, но уже в HSQLDB.

Новые фичи:

	•	SPA-подход вместо server-side rendering;
	•	вход и выход из системы;
	•	защита основной страницы от неавторизованного доступа;
	•	REST-взаимодействие между frontend и backend;
	•	разделение на презентационный слой, API и слой бизнес-логики;
	•	responsive layout для трёх диапазонов ширины экрана;
	•	использование компонентной библиотеки PrimeReact;
	•	более современная организация состояния пользовательского интерфейса;
	•	повторное проектирование предыдущей логики в архитектуре “backend + frontend”.

Как развивается проект от лаборатории к лаборатории

Курс выстроен как последовательное усложнение одной и той же прикладной задачи.

	1.	Lab 1 — базовый web stack: HTML, CSS, JavaScript, Apache, FastCGI, AJAX.
			Основной акцент — понять HTTP, форму, валидацию, серверную обработку и асинхронные запросы.
	2.	Lab 2 — переход к классическому Java web application.
			Появляются сервлеты, JSP, MVC, контейнер сервлетов и более формальная серверная архитектура.
	3.	Lab 3 — переход к компонентному Java EE-подходу.
			Добавляются JSF, managed beans, JPA, ORM и полноценное хранение результатов в PostgreSQL.
	4.	Lab 4 — переход к современной клиент-серверной архитектуре.
			Backend реализуется как REST API на Java EE / EJB, а frontend — как React + Redux SPA с авторизацией и адаптивным интерфейсом.

Таким образом, папка web показывает развитие навыков от базовой веб-разработки и server-side Java до многослойной архитектуры с REST, ORM, базой данных и современным frontend.
